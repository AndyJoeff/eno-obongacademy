Module 1: Introduction to Web Programming

Before coding, let's understand how websites actually render on your screens.

What is the Web?

Every time you visit a website, communication happens between your computer (the Client) and a powerful server machine hosting the physical project files (the Server).

The Three Pillars of Web Development

Front-end projects are supported by three core technologies working in harmony:

  1. HTML: Structural bones.
  2. CSS: Visual skin and colors.
  3. JavaScript: Behavior and functional muscles.
Fig 1.1: Web developers write source code which browsers interpret into clean user interfaces.

Module 2: HTML (HyperText Markup Language)

HTML uses tag formatting arrays to build content segments inside your web pages.

How Tags Function

Tags are surrounded by angle brackets like <p>. Most elements have matching starting and ending markers.

<p>This is a simple paragraph block in HTML.</p>

Audio Supplement: Elements & Nesting Explained

Listen to this quick audio guide explaining how tags reside nested inside one another to construct an HTML DOM tree.

Standard Structural Boilerplate

Every standard webpage follows this basic structure:

<!DOCTYPE html>
<html>
<head>
    <title>My First Webpage</title>
</head>
<body>
    <h1>Welcome to my Academy!</h1>
</body>
</html>

Module 3: CSS (Cascading Style Sheets)

CSS translates black and white tag structures into styled, colorful, layouts.

Understanding Rulesets

CSS selectors target HTML tags to assign custom presentation attributes (properties & values).

/* Styling all body texts */
body {
    background-color: #f7fafc;
    color: #2d3748;
    font-family: Arial, sans-serif;
}

Visual Guide: Modifying CSS Live

Watch this video walk-through demonstrating how changing custom styles updates the viewport and transforms layout structures in real-time.

The Three Implementation Methods

  • Inline: Inserted directly onto individual elements via the style="" parameter.
  • Internal: Included within custom <style> container tags inside the head.
  • External: Managed on distinct .css files linked into head environments.
